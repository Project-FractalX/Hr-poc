VI. Threats to Validity

Internal validity: Latency benchmark uses local Python servers; JVM warm-up, garbage collection, and OS scheduling are not fully controlled. Results are representative of serialization + transport overhead, not production gRPC throughput.

External validity: Evaluation uses two hand-crafted applications; real-world monoliths may have more complex dependency graphs, circular references, or annotation-free code that FractalX's AST analysis does not yet handle.

Construct validity: LOC is estimated from non-blank lines in generated files only; generated scaffolding may inflate this count.

Reproducibility: All source is in scripts/evaluate.py. Re-run with python scripts/evaluate.py --runs 10 for higher confidence.
